🔹 Step 1: Create a New S3 Bucket
1. Login to AWS Console

2. Search for S3 → open S3 Dashboard

3. Click Create bucket

4. Fill details:

   * Bucket name: your-unique-name (e.g., maureen-lab-bucket-123)

   * AWS Region: (Select your region – usually ap-south-1 (Mumbai))

      5. Keep all settings default
 (Block all public access -> keep it ON for now)

      6. Scroll down → Click Create bucket

✔ Your bucket is created.




Step 2: Upload a Sample File
         1. Click on your bucket name

         2. Click Upload

         3. Click Add files

         4. Select any file from your PC (e.g., sample.txt)

         5. Scroll down → click Upload


 Step 3: Enable Versioning
            1. In your bucket → click Properties tab

            2. Scroll to Bucket Versioning

            3. Click Edit

            4. Select Enable

            5. Click Save changes
Step 4: Make One Object Public & Show URL
               1. Go to the Objects tab

               2. Click on the file you uploaded

Step 2: Turn OFF Block Public Access
                  1. Scroll to Block public access (bucket settings)

                  2. Click Edit

                  3. Uncheck (turn OFF) the top option:
✔ Block all public access

                  4. AWS will show a warning → type confirm in the box

                  5. Click Save changes

AWS recently introduced Bucket owner enforced ACL mode.
Fix: Enable ACLs
                     1. Go to your bucket → Permissions tab

                     2. Scroll to Object Ownership

                     3. Click Edit

                     4. If it says:

❌ Object Ownership: Bucket owner enforced (ACLs disabled)
Then change it to:
✔ ACLs enabled → Object writer
                        5. Save changes

                        6. Now go back to Objects → Actions

                        7. “Make public using ACL” will be enabled
